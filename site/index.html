Multi-agent · Local-first RAG · Python + Claude

Watched videos and scattered notes become a [[connected]] knowledge graph - and finished blog posts.

A pipeline of specialized agents extracts the durable ideas from transcripts and notes once, links them into an Obsidian graph, and reuses that graph to draft concept-first posts. Retrieval runs on your own machine; nothing is re-keyed by hand.

The pipeline

Five stages, each its own agent

01

Ingest the sources

Python scripts pull YouTube transcripts by URL or ID, scrape course pages, and convert local knowledge into Markdown - each landing in an inbox, skipping anything already there.

youtube-transcript-api · stdlib scrapers
02

Extract structured notes

A transcript-processing agent turns each raw transcript into a reusable note: the core thesis, key insights with quoted receipts, workflow recipes, and block IDs that link back to the exact moment.

agent brief · fixed note scaffold
03

Connect the knowledge graph

A vault-connectivity agent audits the whole vault for weak links, creates missing concept and hub nodes, merges duplicates, and enforces two-way wikilinks - so ideas surface where they are needed.

Obsidian · wikilinks · block refs
04

Retrieve by meaning

Two retrieval tiers over the same notes: a lightweight ChromaDB index the writer calls in code, and a local hybrid-search server (keyword + vectors + reranking) that Claude Code queries directly.

ChromaDB · fastembed · BM25 + reranker
05

Write the post

The writing agent assembles a content object from what it retrieved and drafts a concept-first, ~700-word post - no code, valid frontmatter, and a call-to-action back to the source video.

Claude · strict output rules
Retrieval

Two tiers, one vault

Programmatic

rag/ - the writer's index

A persistent ChromaDB collection with local, offline embeddings. python -m rag index embeds every note and doc; the writing agent calls rag.search() to ground each draft. No API key, nothing leaves the machine.

Interactive

knowledge-rag - hybrid search in the editor

A local MCP server pairs BM25 keyword search with vector embeddings and cross-encoder reranking. Claude Code queries the same vault natively with search_knowledge() - reranked, and fully local.
